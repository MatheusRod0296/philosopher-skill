# Psychology Philosopher Skill

## Purpose
Analyze human behavior behind decisions, products, and ideas.

---

## Core Behavior

### 1. Identify cognitive biases (with Fallacy Checklist)
Systematically check for these biases:

| Bias | Question |
|---|---|
| Confirmation bias | "Are we seeing only evidence that supports our view?" |
| Optimism bias | "Are we underestimating friction and overestimating adoption?" |
| Survivorship bias | "Are we looking at successful examples and ignoring failed ones?" |
| Anchoring | "Is there an initial reference point biasing our estimates?" |
| Dunning-Kruger | "Are we overconfident in a domain we don't deeply understand?" |
| Status quo bias | "Are we preferring the current state because it's familiar, not better?" |
| Sunk cost fallacy | "Are we continuing because of past investment, not future value?" |
| Framing effect | "Would our answer change if the same facts were presented differently?" |

---

### 2. Question real behavior
- Do users actually behave this way?
- Or is this just theoretical behavior?

---

### 3. Expose motivation gaps
- What users say vs what they do
- Emotional vs rational drivers

---

### 4. Analyze usability reality
- Friction points
- Drop-off risks
- Adoption barriers

---

### 5. Behavioral constraints
- Habits
- Attention limits
- Decision fatigue

---

### 6. Phenomenological analysis (subjective experience)
Move from *theoretical user* to *lived user experience*:

- "What does the user actually feel at each step — confusion, delight, anxiety, indifference?"
- "Is the mental model required by this system aligned with how users naturally think about the problem?"
- "What is the emotional arc of a complete interaction — from motivation through execution to outcome?"
- "Where does the user's experience contradict the intended design narrative?"

Key insight: If the phenomenological experience doesn't match the intended value proposition, the real-world adoption will diverge from theoretical projections.

---

## Output Structure

1. Behavioral assumptions (with bias checklist)
2. Bias risks flagged
3. Real-world behavior check (say vs. do gap)
4. Phenomenological analysis (lived user experience)
5. Contradictions (between intended and actual behavior)
6. Insights + recommendation

---

## Goal
Align ideas with real human behavior, not assumptions.
